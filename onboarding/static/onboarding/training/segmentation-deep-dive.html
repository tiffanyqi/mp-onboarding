Segmentation Deep Dive

Multi-Segmentation

  • Filtering: specifying a property value
  • Segmenting: specifying just the property key
  • Can segment twice
  • Return 625 values in multi-seg
    • Top 25 of the first property and top 25 of the 2nd property combined
    • Undocumented API endpoint because it might be subject to change in a future iteration of segmentation
    • Inner and outer are the two properties you’re segmenting by in multi-seg
    • The limit in the UI is the top 25 for both properties
  • If you segment by something with very high cardinality (city, current level, name), you will run into a case where you have more than 25 values for that property. In that case, you can export it and it will ask you if you want to export the entire dataset or the 625 segments (25 * 25), but the limit there is top 100 (100 * 100 = 10,000).
  • FUN FACT: Segmentation can query for more than a year, but not before July 2011**

Type-Casting

  • Examples for why you may want to typecast - e.g. Integers are bucketed by Mixpanel, so if you want to study specific product/course ids, you will want to typecast the property as a string
  • Reasons you would want to do this: using special properties of data types
  • Changing the BY operator for Integers into SUM, AVG, MAX, MIN, etc.
  • Unix dates (Typecasted, instead of date properties)
    • For when users want to see property values between two specific dates (can be used in People also)
    • Typecast into a number, then copy and paste the timestamps (This is not yet in People, so you have to typecast to an integer and use the timestamps)
    • Country codes → Equals, not contains.

Session Length Tracking

  • GA’s funnels are based on session IDs
  • Our philosophy is much different when it comes to session tracking...we really want to encourage a more insightful metric
  • We don’t track this automatically
  • There’s no industry standard for what a good session length is

Incremental Super properties

  • Event count (# of times) as a super property
    • You have to paste a snippet, read the Community Tip
    • caveat - across devices you need to increment as anticipated rather than starting over
  • What is another example of when we could use incremental super properties?
    • Number of purchases, number of logins, anything that you’re tracking as an incremental people property translates really well to an incremental super property
